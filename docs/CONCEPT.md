# Benchlet Concept

## 背景（何が辛いのか）

* レガシー機器（SSH/Telnet/CLI中心）は、作業の“手順・根拠・再現”が散らばりやすい
* Ansibleによる自動化は進んだが、実行の煩雑さ、ログ確認/比較の手間、品質（パースすり抜け）の切り分けがボトルネック
* オフライン環境で、開発→本番反映が遅く、現場は“その場で確実に回る仕組み”が必要

## 目指す方向

* Carry-in / Carry-out：持ち込める実行環境、持ち帰れる証跡（成果物が必ず残る）
* Toolbox-ready：作業に必要な道具（実行・変数・ログ・比較・判定）が最初から揃う
* Repeatable：誰がやっても同じ導線で同じ成果物に到達（属人性を下げる）
